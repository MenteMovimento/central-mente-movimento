Login, permissões, RLS no Supabase e funções seguras na Vercel.

Utilizador

Entrar na aplicação

  1. Abrir o website publicado.
  2. Introduzir email e password.
  3. Opcionalmente ativar "Lembrar neste dispositivo".
  4. Clicar em "Entrar".

A conta tem de existir no Supabase Auth e estar ativa na lista de utilizadores autorizados.

Utilizador

Painel principal

  • Pesquisar por nome, nº de sócio, NIF, localidade ou email.
  • Ver total de sócios e quotas em atraso.
  • Filtrar por todos, quotas em atraso ou quotas em dia.
  • Ordenar a lista por nome, nº de sócio, localidade ou quotas.

Utilizador

Criar ou editar um sócio

Campos principais

  • Nº de sócio, opcional e único quando preenchido.
  • Nº de Ata de Aprovação, opcional.
  • Nome, morada, código postal e localidade.
  • BI/CC opcional, NIF, categoria, nascimento, telemóvel e email.
  • Observações internas para notas úteis da equipa.

Regras práticas

  • Confirmar sempre o nome antes de guardar.
  • Usar código postal no formato 0000-000.
  • Usar NIF com 9 dígitos.
  • Quando não existir nº atribuído, deixar vazio.
  • Guardar no fim e confirmar que a ficha aparece na tabela.

Utilizador

Quotas

  • A coluna "Quota anual" mostra o último ano pago.
  • Se a quota estiver atrasada, a linha aparece destacada a vermelho.
  • O botão "Pagar quota" avança sempre um ano.
  • O pagamento pede confirmação antes de gravar.
  • A data do pagamento é preenchida automaticamente.

Manutenção diária

Rotina recomendada

  1. Entrada: confirmar login e sessão individual.
  2. Novos sócios: preencher ficha, nº de sócio se existir, nº de ata e data de adesão.
  3. Quotas: filtrar atrasados e usar "Pagar quota" quando houver regularização.
  4. Dados: corrigir contactos, moradas e observações internas quando necessário.
  5. Exportação: exportar Excel regularmente para backup administrativo.
  6. Histórico: consultar quando houver dúvidas sobre alterações.

Manutenção diária

Cuidados para qualquer utilizador

Não apagar sem confirmar

Apagar remove a ficha do sócio. Quando houver dúvida, editar ou adicionar observação interna.

Passwords individuais

Cada pessoa deve usar a sua conta. Não partilhar passwords entre utilizadores.

Computadores partilhados

Terminar sessão depois de usar a aplicação fora de computador pessoal.

Dados pessoais

Recolher apenas dados necessários e manter exportações em local protegido.

Programador

Como atualizar o website

  1. Editar: alterar os ficheiros necessários na pasta do projeto.
  2. Testar localmente: abrir a app num servidor local antes de publicar.
  3. Validar JavaScript: correr node --check app.js.
  4. Guardar no Git: fazer commit com uma mensagem clara.
  5. Enviar para GitHub: fazer push para o branch principal.
  6. Publicação: a Vercel faz redeploy automaticamente.
  7. Confirmar: abrir o site publicado e, se necessário, fazer Ctrl + F5.

Programador

Quando há mudanças na base de dados

Regra principal

Alterar código não chega quando é criado um campo novo. Também é preciso executar o SQL correspondente no Supabase.

Onde executar

  • Entrar no painel Supabase.
  • Abrir o projeto da aplicação.
  • Ir a SQL Editor.
  • Colar o ficheiro SQL da pasta supabase/.
  • Clicar em Run.

Exemplos recentes: add-quota-paid-at.sql e add-approval-minute-number.sql.

Programador

Funções seguras e variáveis

Funções API

  • api/create-user.js cria utilizadores sem ir manualmente ao Supabase.
  • api/delete-user.js elimina utilizadores com proteções adicionais.
  • Ambas confirmam sessão e perfil de administrador.

Variável importante

  • SUPABASE_SERVICE_ROLE_KEY fica apenas na Vercel.
  • Nunca colocar esta chave em config.js.
  • Nunca publicar esta chave no GitHub.

Programador

Checklist antes de publicar alterações

  • O pedido foi compreendido e está limitado ao que é necessário?
  • Os ficheiros alterados são os corretos?
  • O JavaScript passa em node --check app.js?
  • Se houver campo novo, existe SQL em supabase/?
  • O site foi testado localmente no browser?
  • A alteração foi enviada para GitHub?
  • A Vercel publicou a versão nova?
  • O utilizador final sabe se precisa de correr SQL no Supabase?

Passwords e 2FA

Backups

Resumo final

O essencial a memorizar

A app serve para manter os sócios, quotas e acessos organizados num website online protegido.

Utilizadores devem manter os dados certos, exportar backups e evitar apagar sem confirmação.

Programadores devem mexer nos ficheiros certos, testar localmente, publicar pelo GitHub e aplicar SQL no Supabase quando houver mudanças na base de dados.
